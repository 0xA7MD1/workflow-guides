# RULES.md – قواعد الـ Agent

## القواعد الأساسية
1. **اقرأ `DESIGN_SYSTEM.md` أولاً** قبل أي تعديل على التصميم.
2. **لا تستخدم مكتبات خارجية** إلا بموافقة صريحة. يمكن استخدام TailwindCSS و React/Next.js المدمجين.
3. **اللغة الأساسية هي العربية (RTL)**، تأكد من إضافة `dir="rtl"` في `<html>` (في `app/layout.tsx`).
4. **استخدم TypeScript** في كل الملفات.
5. **المكونات التفاعلية** تحتاج `'use client'`. استخدم Server Components للصفحات الثابتة قدر الإمكان.
6. **استخدم TailwindCSS** لتحقيق التصميم. لا تكتب CSS عادي إلا للضرورة القصوى.
7. **المسافات والألوان** تتبع `DESIGN_SYSTEM.md`. أضف أي ألوان جديدة في `tailwind.config.ts` تحت `extend.colors`.
8. **البيانات** تُجلب من `public/workflows.json` باستخدام `fetch` في Server Component أو `useEffect` في Client Component.
9. **التخزين المحلي** يتم عبر دوال في `lib/storage.ts` تغلف `localStorage`.
10. **كل مهمة جديدة** تُضاف أولاً إلى `TASKS.md` قبل تنفيذها.
11. **بعد إكمال أي مهمة**، قم بتحديث `TASKS.md` بوضع `[x]` عليها واكتب ملخصاً بسيطاً للتغييرات.
12. **استخدم `pnpm`** كمدير حزم (وليس npm أو yarn). أي أمر تنفيذ يكون بـ `pnpm run dev`, `pnpm lint`, `pnpm build` إلخ.

## قواعد التصميم
- جميع الأزرار القابلة للنقر يجب أن يكون لها `transition` سلسة.
- عند فتح أي عنصر (مرحلة أو خطوة)، يجب أن يكون هناك `animation` بسيطة.
- شريط التقدم يعتمد على عدد المراحل المفتوحة، وليس الخطوات.
- استخدم متغيرات Tailwind بدلاً من CSS custom properties عندما يكون ذلك ممكناً.
